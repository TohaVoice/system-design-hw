= Функциональные требования
:icons: font

== Advertisement Service
* Создание/редактирование/архив объявлений с мультимедиа
* Поиск и фильтрация (по категориям, цене, локации)
* Система рейтингов и отзывов

== Admin Service
* Интеграции:
** Система хранилища (S3, Yandex Cloud)
** Антивирусное сканирование (ClamAV, VirusTotal API)
** Нормализация качества медиа контента (ImageMagick / FFmpeg)

== Moderation Service
* Модерация отправленного контента (тут тоже возможна статусная модель)

== User & Auth Service
* Регистрация/аутентификация (OAuth2, JWT)
* Управление профилями
* Ролевая модель (покупатель, продавец, модератор)
* История действий*

== Order Service
* Создание заказа
* Просмотр и подтверждение заказов покупателем
* Интеграция с системой платежей
* История заказов
* Управление статусами (Saga Pattern)

.Статусная модель Order Service
[cols="1,3", options="header"]
|===
| Статус | Описание
| Открыт | покупатель создал и отправил заказ продавцу
| Подтвержден | продавец подтверждает заказ и ожидает оплаты
| Оплачен | покупатель оплатил заказ
| В процессе | Покупатель собирает груз, организовывает его отправку через службу доставки, доставка выполняется
| Выполнен | заказ доставлен, покупатель ставит отметку о выполнении заказа. Либо по истечении определенного времени с момента доставки
| Отклонен | Заказ может быть отклонен до подтверждения продавцом или покупателем
| Возврат | может быть оформлен покупателем. По сути приводит к обратным действиям. (Тут вероятней всего будет своя статусная модель, опишу при необходимости)
|===


== Payment Service
* Интеграция с платежными системами (Stripe, Tinkoff, Qiwi)
* Генерация чеков

== Delivery Service
* Карточки доставки
* Интеграция с СДЭК, Почтой России и тд
* Трекинг посылок

== Notification Service
* Email/SMS/Push-уведомления
* Шаблоны сообщений
* Очередь отправки

== Chat Service
* Real-time общение покупатель-продавец
* Прикрепление медиа (??)
* Уведомления о новых сообщениях
